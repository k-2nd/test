Divider Generator IPコア 変更内容要約
• 演算精度の拡張（ビット幅の変更）
• Dividend（被除数）: 26bit から 32bit へ拡張。
• Divisor（除数）: 17bit から 20bit へ拡張。
• これに伴い、出力である m_axis_dout_tdata の幅も 32bit から 40bit（5バイト）へ拡大。
• レイテンシ（遅延）設定の自動化
• Latency Configuration を Manual から Automatic へ変更。
• これにより、計算パイプラインの深さがツールの最適化に任され、実効レイテンシが 16サイクル から 38サイクル へ増加。
• 制御インターフェースの有効化
• ACLKEN / ARESETN: CORDICと同様に、クロックイネーブルとリセット信号を有効化。
• ポートマップに CE ポートが追加され、未接続時の動作維持（High固定）設定を適用。
• 合成フローの最適化
• 合成方式を GLOBAL から OUT_OF_CONTEXT へ変更し、ビルド時間を短縮。
• AXI4-Stream インターフェースの整合
• データ幅の拡張に伴い、TDATA_NUM_BYTES が 4バイト から 5バイト へ更新。
除算器はビット幅を増やすと回路規模と計算時間（レイテンシ）が大きく増える傾向にあります。今回、レイテンシが 16 → 38 と倍以上に増えているため、上位のRTL側でタイミング調整（データの待ち合わせ）が必要になっていないか確認することをお勧めします。
